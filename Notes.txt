Design Principles
------------------

These are the SOLID principles of object-oriented design

Reason for SOLID principles:
- Encourages us to create more maintainable, understandable and flexible software

The below five concepts make up our SOLID principles:

S - Single Responsibility
O - Open-Closed
L - Liskov Substitution
I - Interface Segregation
D - Dependency Inversion

Single Responsibility
---------------------
- A class should only have one responsibility
- It should only have one reason to change

Benefits
- Class with one responsibility will have fewer test cases
- Less functionality in a single class will have fewer dependencies
- Smaller well organized classes are easier to search

public class Book {
    private String name;
    private String author;
    private String text;
    // constructors, getters and setters
    public String replaceWordInText(String word, String replacementWord) {
        return text.replaceAll(word, replacementWord);
    }
    public boolean isWordInText(String word) {
        return text.contains(word);
    }
}

Open for Extension and Closed for Modification
----------------------------------------------
- Classes should be open for extension but closed for modification
- When fixing bugs, we can modify the code

Let’s explore the concept with a quick code example. As part of a new project, imagine we’ve implemented a Guitar class.

public class Guitar {
    private String make;
    private String model;
    private int volume;
}

Now we decided to add a cool pattern to make it look more rock and roll
